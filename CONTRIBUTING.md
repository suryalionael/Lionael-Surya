# Contributing

This is a personal portfolio repository, but it's maintained like a real engineering project — feedback is genuinely welcome.

## How to Suggest Changes

- **Found a bug, broken link, or factual error?** Open a GitHub Issue describing it.
- **Have a suggestion on architecture, project scope, or technology choice?** Open an Issue or Discussion — I'm a student building this specifically to get feedback from working engineers.
- **Spotted something a recruiter/hiring manager would flag?** That feedback is the most valuable kind — please don't hold back.

## Commit Conventions

This repo follows [Conventional Commits](https://www.conventionalcommits.org/):

| Prefix | Use for |
|---|---|
| `feat:` | A new project or feature within a project |
| `fix:` | Bug fixes |
| `docs:` | README/documentation changes |
| `chore:` | Tooling, dependency, or config changes |
| `refactor:` | Code restructuring without behavior change |

## Project Structure Rules

Every project folder must contain, at minimum, before being marked anything other than 🔜 Planned:

1. A `README.md` with a stated business problem, objective, architecture, and tech stack.
2. Reproducible setup instructions (a `requirements.txt`, `docker-compose.yml`, or equivalent).
3. No hardcoded secrets or credentials — use `.env.example` files and document required environment variables.

## Code of Conduct

Be direct, be specific, and assume good faith. Technical critique is welcome; it's how this portfolio improves.
